# ダニエルズ式トレーニングペース算出仕様書 (DANIELS_FORMULA_LOGIC)

## 0. 各ペースの定義と算出係数（サマリー）

* **E-Pace**: Tペースの1.25倍（回復と土台）

* **T-Pace**: 10kmペースの1.022倍（30-40分維持できる「心地よいきつさ」）

* **I-Pace**: 5kmペースの1.00倍（VO₂maxを刺激する「きつい」領域）

* **R-Pace**: 1500mペースの1.00倍（スピードと経済性）

## 1. 算出の仕組み (Data Flow)

本システムは、`user_stats.json` に記録されたユーザーの走力に基づき、`vdot_table.csv` から最適なトレーニング強度を動的に導出します。

1. **走力の特定**: `user_stats.json` の `base_distance`
と `base_time` をキーとして使用。
2. **VDOT行の抽出**: CSVから、入力されたタイム以下で最も速い（走力に近い）VDOT行を特定。
3. **基準値の取得**: 特定した行から、計算の種となるレースタイム（5km, 10km）を取得。
4. **ロジック適用**: 下記のアルゴリズムに基づき、T/I/R の各強度を算出。

## 2. 算出仕様書 (DANIELS_FORMULA_LOGIC)

### E-Pace (Easy Run / Recovery)

* **目的**: 「有酸素ベース」の構築と「回復促進」
* **定義**（第4版 p.100）: ランニングの土台を作る最も基本的な強度。心筋の強化、筋肉への血液供給（毛細血管の開発）、および脂質代謝の改善を促します。
* **計算式**: (Threshold Pace) × 1.25 （※秒/kmを1.25倍して遅くする）検証 (VDOT 44):Tペース = 4:43/km (283秒)$283 \times 1.25 = 353.75$ 秒E上限 = 5:54/kmロジックの根拠: Daniels公式では、Easy paceは「最大心拍数の65～79%」の範囲。Thresholdペースより約25%程度遅く設定することで、翌日のQセッションへ活力を繋げつつ、怪我のリスクを最小限に抑えて走行距離を稼ぐことが可能になるため。

### T-Pace (Threshold / Tempo)

* **目的**: **「乳酸作業閾値（LT）」の向上**
* **定義（第4版 p.103）**: **「心地よいきつさ（Comfortably Hard）」**
* 「今のペースを30～40分間維持しろと言われたらできるかどうか」を基準とする。
* **計算式**: `(10kmレースペース) × 1.022`
* **ロジックの根拠**: 10kmを全力で走るペース（約45〜50分持続）よりも、さらに「維持できる確信」が持てるわずかに緩やかな強度。血中の乳酸が急増する直前のポイントを刺激するため。

### I-Pace (Interval / VO₂max)

* **目的**: **「最大酸素摂取量（VO₂max）」の向上**
* **定義**: **「きつい（Hard）」**
* 閾値を超えた「きつい」領域。10〜12分程度しか維持できない強度。
* **計算式**: `(5kmレースペース) × 1.00`
* **ロジックの根拠**: 5kmの全力疾走ペースは、VO₂maxを最大限に刺激する強度に相当するため。

### R-Pace (Repetition)

* 目的: 「ランニング経済性」と「スピード」の向上
* 定義（第4版 p.135）: スピード養成のためのインターバル。
* 計算式: (1500mレースペース) × 1.00
* 検証 (VDOT 51):1500m = 5:18 (318秒)キロペース = 212秒/km400m換算 = $212 \times 0.4 = 84.8$ 秒（約 85〜86秒）200m換算 = $84.8 \div 2 = 42.4$ 秒（約 43秒）検証 (あなたの走力 VDOT 44):1500m = 6:03 (363秒)キロペース = 242秒/km400m換算 = 96.8 秒（1分37秒）200m換算 = 48.4 秒ロジックの根拠: Rペースは「約 6分間全力で維持できるペース」に基づき設定されるため、1500m（約 5〜6分の競技）のレースペースと等しくなる。

## 3. Qセッションの戦略的役割 (Phases)

各フェーズで重視する強度の組み合わせにより、段階的に走力を完成させます。

| フェーズ | フォーカス | 主な狙い |
| :--- | :--- | :--- |
| **Phase II** | **R (Repetition)** | **スピードの土台作り。** 十分な休息を取りながら、速い動きに慣れ、効率的なフォームを構築する。 |
| **Phase III** | **I (Interval)** | **エンジンの最大化。** VO₂maxを刺激し、心肺能力の天井を最も高く引き上げる時期。 |
| **Phase IV** | **T (Threshold)** | **持久力の完成。** レースで維持できる巡航速度を安定させ、目標タイム達成を確実にする。 |

## 4. エンジニアリング・ノート

* **逆転の回避**: レースペースに対して「速くする（係数<1）」のか「遅くする（係数>1）」のかを、生理学的特性（Tは10kmより遅い、Iは5kmと同等など）に基づき厳密に定義しました。
* **VDOTとの整合性**: 係数（1.022, 0.83等）は、ダニエルズのVDOTテーブル第4版の数値から逆算して導出しており、書籍の推奨値と高い精度で一致します。
* **動的計算の安全性**: PB（自己ベスト）が更新された際、`user_stats.json` を書き換えるだけで、常に「その時点の最適負荷」が再計算されます。
* **単位の統一**: すべての計算は「秒/km」に変換して行い、最終出力時にのみ `MM:SS` 形式へ整形することで、計算誤差を最小限に抑えています。
