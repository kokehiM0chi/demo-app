# ダニエルズ式トレーニングペース算出仕様書 (DANIELS_FORMULA_LOGIC)

## 1. 算出の仕組み (Data Flow)

本システムは、`user_stats.json` に記録されたユーザーの走力に基づき、`vdot_table.csv` から最適なトレーニング強度を動的に導出します。

1. **走力の特定**: `user_stats.json` の `base_distance` と `base_time` をキーとして使用。
2. **VDOT行の抽出**: CSVから、入力されたタイム以下で最も速い（走力に近い）VDOT行を特定。
3. **基準値の取得**: 特定した行から、計算の種となるレースタイム（1.5km, 5km, 10km）を取得。
4. **ロジック適用**: 下記のアルゴリズムに基づき、T/I/R の各強度を算出。

---

## 2. メニュー別：トレーニングの目的と算出ロジック

各セッションの「目的」と、それに基づいた「計算根拠」の対応表です。

### 閾値走 (T: Threshold)

* **目的**: **「乳酸作業閾値（LT）」の向上**
* 血中の乳酸を再利用・処理する能力を高め、レース後半の粘り強さを養います。
* **計算式**: `(10kmレースペース) × 1.07`
* **ロジックの根拠**: 10kmレースペースより約7%負荷を下げた強度が、ダニエルズ理論における「20〜30分維持できる快適なきつさ」に相当するため。

### インターバル (I: Interval)

* **目的**: **「最大酸素摂取量（$VO_2max$）」の向上**
* 心肺機能を高め、筋肉により多くの酸素を送り込めるようにし、有酸素能力の天井を引き上げます。
* **計算式**: `(5kmレースペース) ÷ 1.0`
* **ロジックの根拠**: 5kmの全力疾走ペース（約15〜20分維持可能）が、ダニエルズ理論における $VO_2max$ 強化の強度とほぼ合致するため。

### レペティション (R: Repetition)

* **目的**: **「ランニング経済性」と「スピード」の向上**
* 神経系を刺激し、効率の良いフォームを身につけます。同じペースでもエネルギー消費を抑えて走れるようになります。
* **計算式**: `(1.5kmレースペース) × 0.95` （を200m/400mに換算）
* **ロジックの根拠**: 1500m走よりも5%速い「キレ」を重視したスピード設定を行うことで、無酸素作業能とランニングの効率性を高めるため。

---

## 3. Qセッションの戦略的役割 (Phases)

各フェーズで重視する強度の組み合わせにより、段階的に走力を完成させます。

| フェーズ | フォーカス | 主な狙い |
| :--- | :--- | :--- |
| **Phase II** | **R (Repetition)** | フォームの効率化。まずは速い動きに慣れる。 |
| **Phase III** | **I (Interval)** | エンジン（心肺能力）の最大化。最も負荷が高い時期。 |
| **Phase IV** | **T (Threshold)** | スタミナの完成。レースで維持できる速度の底上げ。 |

---

## 4. エンジニアリング・ノート

* **動的計算のメリット**: `user_stats.json` を更新（PB更新など）するだけで、全24週間のプランが自動的に新しい走力に最適化されます。
* **実装上の工夫**: `timedelta` を用いて、`MM:SS` 形式の文字列と数値演算を安全に行き来させています。
